# 鸭腿叔叔

一个微信聊天框形式的虚构经营小游戏。玩家以“鸭腿叔叔”的第一视角经营校园烤腿生意，在群聊、供应商、家人、舆论和监管消息之间做选择。

![鸭腿叔叔头像](assets/duck-leg-uncle-avatar.png)

## 作品概念

这个项目不是传统的经营面板游戏，而是把“聊天界面”本身变成玩法。玩家面对的不是抽象按钮，而是一串不断出现的群消息、供应商私聊、家人提醒、平台通知和舆论截图。

每一次回复都会影响经营状态：

- 收益：现金、毛利空间、成本和日常花销。
- 风险：票据缺口、监管压力、良心亏空。
- 流量：群聊热度、口碑、未读压力。

回复太慢时，群聊和舆论不会等玩家，红点会继续堆积，事件也会自己发酵。

## 游戏机制

游戏以 7 天经营周期推进。每一天会出现多个任务，玩家需要在不同聊天对象之间切换，阅读消息并做出回复。

核心机制包括：

- **聊天列表驱动任务**：任务不是以关卡按钮出现，而是以微信群、供应商、家人、平台等会话出现。
- **时间压力**：营业时间从 09:00 推进到 21:00，未处理红点会带来连续压力。
- **多指标联动**：短期赚钱、口碑维持、票据合规和舆论风险互相牵制。
- **状态记忆**：前期是否明示品类、是否用过灰色货源、是否退款或公关，会影响后续文本与结局分析。
- **多结局解释**：结局会结合最终收益、风险、流量和关键选择，给出经营复盘。

## 为什么用聊天框

校园生意、群聊传播和舆论风险天然是信息流问题。聊天框适合表达这种“同时被很多人催促”的状态：

- 群聊可以制造热度和误解。
- 供应商私聊可以带来利润和隐患。
- 家人提醒可以提供道德压力。
- 平台和监管消息可以把前面的选择串起来。

玩家不需要学习复杂操作，只需要像日常读消息一样推进剧情；但每一句回复都在改变经营账本。

## 运行方式

这是静态页面项目，直接打开 `index.html` 即可运行。

也可以用任意静态服务器部署仓库根目录：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 文件结构

| 文件 | 作用 |
| --- | --- |
| `index.html` | 页面结构，包含开场弹窗、聊天面板、指标栏和设置面板。 |
| `styles.css` | 微信式聊天界面、经营指标、设置抽屉和结局页样式。 |
| `game.js` | 剧情数据、状态机、时间推进、选择效果和渲染逻辑。 |
| `assets/duck-leg-uncle-avatar.png` | 玩家头像和项目主视觉。 |
| `vercel.json` | 静态部署配置。 |

## 代码入口

- `initialState`：定义现金、口碑、良心、风险、票据、流量等初始状态。
- `chapters`：定义 7 天经营剧情、聊天消息和选项。
- `applyEffects`：根据选择更新经营指标。
- `applyOverduePressure` / `applyContinuousPressure`：处理未读红点和时间压力。
- `renderBanners` / `renderConversation` / `renderEnding`：渲染聊天列表、对话和结局。
